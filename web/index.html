Multicluster

In this website we will describe the different Multi Cluster configurations. From a simple 2 kubernetes cluster configration to a more complex one running in a multi-cloud (aws/Google/Azure) multi-scheduler (Kubernetes/Mesos) scenario.

This website runs in 2 different clusters (hosted in EU West and US East) in 2 kubernetes clusters (version 1.8.8 and 1.9.7 with Istio v.0.7.1).

You can find more info in the Github repo

How does it work?

The basic architecture is fairly simple: 1 global load balancer, one kubernetes cluster and that's pretty much it. Once you have this setup, you can add more clusters, more applications.

Is this cluster Federation?

No. A cluster federation has a different philosophy. With multicluster the aim is minimise the operability costs and increase the agility of a platform. Technically, a federation needs a control plane, where the state of the different cluster is kept and managed. In multicluster each cluster is independent. Once the traffic gets into a cluster, it stays there.
